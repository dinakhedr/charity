4. Soft-Delete & Audit Trail Lifecycle 
                stateDiagram-v2
                    [*] --> Active: Record Created (IsDeleted=FALSE)
                    Active --> Active: Edit → ModifiedAt/By
                    Active --> SoftDeleted: User with Delete Permission
softDelete()
                    
                    SoftDeleted --> HiddenFromView: Default filters (IsDeleted=TRUE hidden)
                    SoftDeleted --> VisibleDeleted: Admin toggles "Show Deleted"
                    VisibleDeleted --> Active: Restore action (clears flags)
                    
                    Active --> AuditLog: CREATE/EDIT logged
                    SoftDeleted --> AuditLog: DELETE/RESTORE logged
                    
                    state Active {
                        [*] --> IsDeleted_FALSE
                        DeletedAt_NULL
                        DeletedBy_NULL
                    }
                    state SoftDeleted {
                        [*] --> IsDeleted_TRUE
                        DeletedAt_Timestamp
                        DeletedBy_UserEmail
                    }
Deleted row (red tint)
Active record Full AuditLog sheet stores every mutation

8. Authentication Sequence (Google OAuth + Token Caching) 
                sequenceDiagram
                    participant User
                    participant Page as HTML Page
                    participant Auth as auth.js
                    participant LS as localStorage
                    participant SS as sessionStorage
                    participant Google as Google Identity
                    participant Sheets as Sheets API
                    
                    User->>Page: Open any module
                    Page->>Auth: initPageAuth()
                    Auth->>LS: Read google_token
                    alt Token missing
                        LS-->>Auth: null
                        Auth->>User: redirect to Home.html
                    else Token exists
                        Auth->>Page: onLoadLocal(email) [render cached data]
                        Auth->>Google: initGapiClient()
                        Auth->>SS: check cached access_token
                        alt Valid token
                            SS-->>Auth: valid
                            Auth->>Page: onReady(spreadsheetId, token)
                            Page->>Sheets: authorized API calls
                        else Expired
                            Auth->>Google: request new token (silent)
                            Google-->>Auth: fresh token
                            Auth->>SS: cache token
                            Auth->>Page: onReady()
                        end
                    end

10. Record Creation Data Flow (Example: New Transaction) 
                flowchart TD
                    START[User clicks Add] --> MODAL[Open modal with dynamic lookup dropdowns]
                    MODAL --> FILL[Fill fields & select Category/Donor]
                    FILL --> SUBMIT[Click Save]
                    SUBMIT --> VALIDATE{validateRequired}
                    VALIDATE -->|Missing fields| ERROR[Highlight + toast warning]
                    ERROR --> FILL
                    
                    VALIDATE -->|Valid| SERIALIZE[serializeForm → object]
                    SERIALIZE --> BUILD[Attach metadata]
                    BUILD --> GEN_ID[generateRecordID 8-char]
                    GEN_ID --> TIMESTAMP[getCurrentTimestamp]
                    TIMESTAMP --> USER_EMAIL[CreatedBy = session email]
                    USER_EMAIL --> SET_DELETE[IsDeleted = FALSE]
                    
                    SET_DELETE --> APPEND[appendRow to Sheet]
                    APPEND --> AUDIT[Write to AuditLog: CREATE]
                    AUDIT --> REFRESH[Refresh table view]
                    REFRESH --> TOAST[showToast success]
                    TOAST --> CLOSE[Close modal, clearForm]
